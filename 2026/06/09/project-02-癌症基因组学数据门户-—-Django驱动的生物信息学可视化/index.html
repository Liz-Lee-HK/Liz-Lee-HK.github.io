癌症基因组学数据门户 — Django驱动的生物信息学可视化平台

作者: Liz Lee
发表于2026-06-10
更新于2026-06-10
Projects
BioinformaticsData VisualizationDjangoGenomicsMySQLSurvival Analysis

项目概述

一个基于Django框架的癌症基因组学数据管理与可视化平台，支持患者临床数据管理、突变基因数据库查询、Kaplan-Meier生存分析以及多种基因组学数据可视化（热图、OncoPlot等）。

背景与动机

癌症基因组学研究产生了海量的多维度数据——从患者的临床表型到基因突变频谱、从拷贝数变异到生存预后。研究者需要一个统一的平台来管理、查询和可视化这些数据，才能从纷繁的数据中发现有意义的生物学规律。

技术方案

数据模型设计

  • 患者模型：记录人口学信息（年龄、性别）、临床数据（病理类型、分期）和采样信息
  • 突变基因模型：整合MutSig突变频率数据与OncoKB癌症基因注释，记录每个基因的突变类型、频率和临床意义
  • 生存数据模型：OS（总生存期）月数跟踪，支持Kaplan-Meier生存曲线计算

可视化与分析

  • 基于matplotlib和seaborn实现生存曲线、年龄分布、性别分布、病理类型等统计图表
  • OncoPlot突变全景图展示基因-样本突变矩阵
  • 数据表浏览支持分页查询与多条件筛选
  • Session-based数据探索流程，允许用户在不同分析视图之间切换

工程架构

  • 后端：Django 3.2 + MySQL + uWSGI（生产环境）
  • 科学计算栈：NumPy, pandas, scipy, matplotlib, seaborn
  • 预留PyTorch集成接口，为后续深度学习分析（如病理图像分类）做准备

项目成果

  • 实现了从基因组学原始数据到交互式可视化分析的完整闭环
  • 构建了结构化的突变基因-临床表型关联数据库
  • 平台具备用户认证、文件管理、权限控制等生产级功能
  • 为后续引入病理切片AI分析提供了可扩展的架构基础
