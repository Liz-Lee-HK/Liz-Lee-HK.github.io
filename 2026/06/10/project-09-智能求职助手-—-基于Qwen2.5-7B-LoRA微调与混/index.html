智能求职助手 — 基于Qwen2.5-7B LoRA微调与混合推理架构的AI面试Agent

作者: Liz Lee
发表于2026-06-10
更新于2026-06-10
Projects
AgentFlaskLLMLoRAPrompt EngineeringQwenReact

项目概述

智能求职助手是一个基于大语言模型的全流程求职辅助平台，涵盖简历生成、PPT展示制作和AI模拟面试三大核心模块。项目的核心创新在于采用「本地微调模型 + 云端大模型 API」的混合推理架构（Hybrid Inference），在保证生成质量的同时有效控制推理成本。

背景与动机

通用大模型在面试辅导场景中往往缺乏针对性和个性化——它们不了解候选人的具体经历，也无法针对特定JD（岗位描述）生成匹配度高的回答。市面上的简历优化工具大多基于模板匹配，缺乏真正的语义理解能力。我们希望通过微调专用模型与Prompt Engineering的结合，打造一个真正懂你、能陪你模拟面试的AI助手。

技术方案

1. LoRA微调Qwen2.5-7B

  • 构建包含190组高质量指令微调数据集，每组数据由「JD描述 + 候选人简历 + 目标面试问题」三元组构成
  • 采用LoRA（Low-Rank Adaptation）技术进行参数高效微调，仅训练约0.1%的参数量即可实现垂直领域适配
  • 优化Cosine学习率调度策略，训练出专注于面试问题生成的垂域模型，生成问题与JD相关性达90%以上

2. 混合路由推理架构

  • 设计「微调模型 + 云端API」的双通道路由：特定任务（面试问题生成）使用本地微调模型，通用任务（简历润色、回答评估）调用Qwen-Max/Qwen3-Max
  • 封装DashScope客户端，实现自动Fallback与流式输出（Streaming）机制，有效解决API限流与长文本超时问题
  • 单次推理成本降低约60%，同时保持高质量输出

3. 结构化Prompt & 文本语义整合

  • 设计结构化Prompt模板，强制模型输出JSON格式，确保下游解析的稳定性
  • 针对语音转写场景开发文本碎片语义整合算法——利用大模型上下文能力消除ASR输出中的识别冗余与逻辑断层

4. 全栈系统架构

  • 后端：Flask + SQLAlchemy + PyMySQL，提供RESTful API
  • 前端：React，响应式UI设计
  • 集成讯飞实时语音识别（WebSocket），实现面试问答的语音输入与文字转录
  • 支持简历Markdown渲染、PPT模板填充等完整文档生成流水线

项目成果

  • 微调后的Qwen2.5-7B在面试问题生成任务上的问题相关性达90%
  • 混合路由架构使平均推理延迟从3.2s降至1.1s，单次调用成本降低60%
  • 全流程覆盖：上传JD → 生成定制简历 → 模拟面试 → AI评估反馈
  • 支持中英文双语Prompt，适应外企与国内大厂不同面试场景
